# Public-release checklist

Before manuscript submission, complete these repository-side steps:

1. Confirm that both GitHub repositories are public and that their default
   branch is `main`.
2. Create a tagged release from the exact commits used for the manuscript.
3. Archive the code and dataset releases in Zenodo, Figshare, or an
   institutional repository that assigns persistent DOIs.
4. Add the two archive DOIs to `AVAILABILITY_STATEMENTS.md`, the manuscript,
   and the GitHub repository descriptions.
5. Verify the installation instructions in a fresh directory or on a separate
   machine.
6. Confirm that the SOC-EIS citation, CC BY 4.0 terms, and author
   acknowledgment remain visible in the dataset repository.
